Chapter 3: Adding Style with CSS

CSS

CSS or Cascading Style Sheets is a stylesheet language used to describe the look and feel of websites to your browser. While CSS can be embedded in HTML, its preferred that .css files be designed and linked to the webpages that will use them. Doing so unlocks much greater potential for consistency and automation than embedding alone.

CSS Inheritance

CSS works on an inheritance model where each child element inherits the properties of the element above it. This means that style applied to headers or body elements could be overridden by styles applied to specific classes within those elements (.e.g - style applied to <div class="special_class"> would take precedence over style applied to all <div> elements).

CSS Consistency

Another wonderful aspect of CSS is its ability to enforce consistent theming across your website or project with only a few declarations. Stand-alone stylesheet files can be referenced by more than one HTML document, meaning that any styles used in a project can be applied to the entire project (or specific sections / pages) as desired.

This focus on consistency combines several highly desirable features for any programming language centered around avoiding repetition. Repetition is bad for programmers because it leads to a higher risk of human error (greater entropy), less consistent code (re-creating the same "tool" multiple times), and more time spent debugging. The more streamlined and understandable the code, the easier it is to update and maintain. This is especially true where multiple developers may be working on the same projects.

CSS and the Box

CSS has significant implications for the box model of HTML as well. Using stylesheets, you can modify several aspects of the standard box, including:

  • margin (space between boxes
  • border (border surrounding the padding and content)
  • padding (space between the content and the border)
  • content (the actual HTML content that appears on the website)
