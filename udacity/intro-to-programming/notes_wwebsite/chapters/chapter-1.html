Chapter 1: Fundamentals of the Web and HTML

Web "Design"

The web is comprised of servers and end-hosts that communicate with each other over standard protocols. Most web traffic traverses either the HTTP (port 80) or HTTPS (port 443) protocols. These are the protocols you use to communicate with sites like google (HTTP) or more secure websites like your online banking portal (HTTPS).

In either case, when you enter a URL into your browser, it contacts a webserver with a request for the page you requested. These requests are sent and received using the HTTP protocol, and tell the server to send the correct HTML page back to your browser for viewing. This HTML is then interpreted by your browser and displayed as a webpage. You can view a brief overview with links for further reading at this site.

HTML

HTML or Hypertext Markup Language is a type of document that comprises most webpages on the internet. The HTML standard is made up of two components - Content which is the information you want to display, and Markup which is how that content is formatted. You can see a more in-depth explanation here. This is also a great resource for diving in and learning the basics of any language, not just HTML.

Tags and Elements

HTML is comprised of two types of formatting - tags and elements. This link gives a good description of the difference. Basically, HTML tags are formatting tags, written in the <tag> syntax, while elements are sets of tags with text or other content that they modify. For instance this would be an HTML element written as <em>this</em> in HTML.

Why Computers are Stupid

We say that computers are stupid because they interpret our instructions literally. What this means is that from a binary, logical point of view they do exactly what we tell them to do. This is frustrating for humans who can make logical inferences, contextualize meaning, and use other higher-order functions to interpret what we see and experience. In the future (and even in the present, if this guy gets his way..), computers may have this capacity too.

Inline vs Block Elements

HTML elements come in two varieties - inline and block. Block elements create their own container within which further inline elements can be added. Inline elements can be placed inside these containers to further assist in formatting. Good examples of these are the <p> tag, which forms a block element and the <br> tag, which forms an inline element.
