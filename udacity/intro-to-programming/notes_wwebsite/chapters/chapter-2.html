Tree-like Structure of HTML

All HTML documents are structured based on the DOM (Document Object Model) Model. This model outlines a tree-like document structure that is easy to parse using development tools or apply styles to using CSS.

With HTML, the structure is composed of elements like <div>, <h1>, and even inline elements like <span> which create containers ("boxes") around content. These "boxes" are then grouped into parent/child relationships where elements exist within the context of the elements directly above them, and can inherit properties and styles from those higher level elements. This combination of a heirarchal document structure with style inheritance allows consistent styling to be applied to like elements while also providing the ability to easily navigate the entire document tree and review individual elements.

The Box

Everything in an HTML document is a box. This is a core design principle that helps to segment designs. By splitting everything on the page into boxes and deciding what content should go in each box, the developer can create a structured document that is easy to re-arrange and re-design as needed.

Div Tags and Classes

One very important and very powerful feature of HTML and the box model is the ability to assign classes to elements such as <div>. While <div> elements are a core feature of the box model mentioned earlier, classes allow for style (CSS) to be applied to specific elements based on named attributes specified by the programmer.
